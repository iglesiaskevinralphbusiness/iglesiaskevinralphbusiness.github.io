Mouse Events

Click Event

<div id="app">
	<button v-on:click="increase">Increase</button>
	<p>{{ counter }}</p>
</div>
new Vue({
	el: '#app',
	data: {
		counter: 0
	},
	methods: {
			increase: function(){
				this.counter++;
			}
	}
});

Mousemove Event

<div id="app">
	<p v-on:mousemove="updateCoordinates">Coordinates: {{ x }}/{{ y }}</p>
</div>
new Vue({
	el: '#app',
	data: {
		x: 0,
		y: 0,
	},
	methods: {
			updateCoordinates: function(event){
				this.x = event.clientX;
				this.y = event.clientY;
			}
	}
});

Mousemove Event with Modified Event in Child

This means that child element with not affected by the mouse move event.

<div id="app">
	<p v-on:mousemove="updateCoordinates">Coordinates: {{ x }}/{{ y }} <span v-on:mousemove.stop="">Stop Event<span/></p>
</div>

Properties

Computed Properties Vs Methods Properties

If you declared a rendering function in the page:

Computed will execute the function only if value of related data variable whithin that function is changed.

Method will always run every time data variable are changed. (Disadvantage if your functions code is too long script which is executed but not related on changes, it only interupt the speed performance of app)

Calling function:

Computed called without parenthesis

Method called with parenthesis

<div id="app">
		<button v-on:click="counter1++">Increase</button>
		<button v-on:click="counter1--">Decrease</button>
		<button v-on:click="counter2++">Increase Counter 2</button>
		<p>Counter 1: {{ counter1 }}</p>
		<p>Counter 2: {{ counter2 }}</p>
		<p>Result: {{ renderFlag }} | {{ renderCounter() }}</p>
	</div>
new Vue({
	el: '#app',
	data: {
			counter1: 0,
			counter2: 0,
			flag: ''
	},
	computed: {
			renderFlag: function(){
				console.log('Computed Triggered'); // will not triggere if you click the Increase Counter 2 button
				return this.counter1 > 5 ? 'Greater than 5' : 'Less than 5';
			}
	},
	methods: {
			renderCounter: function(){
				console.log('Method Triggered');
				return this.counter1;
			}
	}
});

Watch Properties

Will be executed when a specific data variable is changed.

new Vue({
			el: '#app',
			data: {
				counter: 0,
			},
			watch: {
				counter: function(){
					console.log('Watch Triggered');
				}
			},
		});

Dynamic Styling with CSS Classes

Basic / Active & Innactive

A box that when you click, it change it state to active and inactive.

.box {
	width: 100px;
	height: 100px;
	margin: 10px;
	background-color: gray;
}
.red {
	background-color: red;
}
<div id="app">
	<div
		class="box"
		@click="active = !active"
		:class="{'red': active}"
	></div>
</div>
new Vue({
	el: '#app',
	data: {
			active: false
	},
});

Multiple Class as Object

.box {
	width: 100px;
	height: 100px;
	margin: 10px;
	background-color: gray;
}
.red {
	background-color: red;
}
.red-border {
	border: 5px solid pink;
}
<div id="app">
	<div
		class="box"
		@click="active = !active"
		:class="{'red': active, 'red-border': active}"
	></div>
</div>

Variable as Class Names

.box {
	width: 100px;
	height: 100px;
	margin: 10px;
	background-color: gray;
}
.pending {
	background-color: red;
}
.cancelled {
	background-color: orange;
}
.done {
	background-color: green;
}
<div id="app">
	<button @click="status = 'pending'" :disabled="status == 'pending'">Pending</button>
	<button @click="status = 'cancelled'" :disabled="status == 'cancelled'">Cancel</button>
	<button @click="status = 'done'" :disabled="status == 'done'">Done</button>
	<div
		class="box"
		:class="status"
	></div>
</div>
new Vue({
	el: '#app',
	data: {
			status: ''
	},
});
